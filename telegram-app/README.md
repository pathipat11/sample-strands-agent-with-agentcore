# Telegram Adapter

A Telegram bot that connects to AWS AgentCore Runtime, enabling personal AI agent access via Telegram Bot API.

## Features

- **Text, image, and document input** -- send photos, PDFs, DOCX, XLSX, CSV for multimodal analysis
- **Rich output** -- Markdown rendered as Telegram HTML, inline code blocks, S3 images delivered as photos
- **Research reports** -- long agent outputs sent as styled HTML files (with embedded charts)
- **File artifacts** -- PPTX, DOCX, XLSX generated by agents delivered via `sendDocument`
- **Human-in-the-loop** -- agent interrupt prompts shown as inline Approve/Decline buttons
- **Model selection** -- `/model` command to switch between Claude models mid-conversation
- **Session reset** -- `/reset` to start a fresh conversation
- **Message debouncing** -- sequential messages within 600ms are combined into one agent request
- **API throttling** -- `@grammyjs/transformer-throttler` for Telegram rate limits
- **Access control** -- optional allowlist by Telegram user ID

## Commands

| Command  | Description                              |
|----------|------------------------------------------|
| `/reset` | Clear session and start a new conversation |
| `/model` | Choose a model (Sonnet 5 / Haiku 4.5 / Opus 4.6) |

## Deployment

### Prerequisites

1. Create a bot via [BotFather](https://t.me/BotFather) and save the token.
2. Get your Telegram user ID (send `/start` to [@userinfobot](https://t.me/userinfobot)).

### Terraform

The adapter deploys as an ECS Fargate service. Enable it by setting variables in `infra/environments/dev/`:

```bash
cd infra
./scripts/deploy.sh \
  -var="enable_telegram=true" \
  -var="telegram_bot_token=<YOUR_BOT_TOKEN>" \
  -var="telegram_allowed_user_ids=<YOUR_USER_ID>"
```

Or add to `terraform.tfvars`:

```hcl
enable_telegram           = true
telegram_bot_token        = "<YOUR_BOT_TOKEN>"
telegram_allowed_user_ids = "<YOUR_USER_ID>"
```

Multiple user IDs can be comma-separated: `"123456789,987654321"`.

### Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `TELEGRAM_BOT_TOKEN` | Yes | BotFather token |
| `RUNTIME_INVOCATION_URL` | Yes | AgentCore Runtime endpoint |
| `COGNITO_TOKEN_URL` | Yes | Cognito M2M token endpoint |
| `M2M_CLIENT_ID` | Yes | Cognito client ID |
| `M2M_CLIENT_SECRET` | Yes | Cognito client secret |
| `DEDUP_TABLE_NAME` | Yes | DynamoDB table for dedup |
| `ARTIFACT_BUCKET` | No | S3 bucket for file artifacts |
| `OWNER_USER_ID` | No | Override user ID sent to agent |
| `ALLOWED_USER_IDS` | No | Comma-separated Telegram user IDs |
| `LOG_LEVEL` | No | `info` (default), `debug`, `warn` |
| `HEALTH_PORT` | No | Health check port (default: 8080) |

### Local Development

```bash
cd telegram-app
npm install
cp .env.example .env  # fill in values
npm run dev
```
